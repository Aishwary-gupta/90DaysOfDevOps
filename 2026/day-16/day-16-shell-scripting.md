## Objective

Today I started learning Bash Shell Scripting, one of the most important skills for every DevOps Engineer. Instead of running commands manually, I learned how to automate repetitive tasks using shell scripts.

---

## Concepts Learned

- Shebang (`#!/bin/bash`)
- Creating executable scripts
- Variables
- User input using `read`
- `echo` command
- If-Else conditions
- File existence checks
- Checking service status with `systemctl`
- Bash debugging using `bash -x`

---

## Commands Used

```bash
touch
nano
chmod +x
echo
read
systemctl
bash -x
```

---

## Key Learnings

- Shell scripts automate repetitive Linux tasks.
- The shebang tells Linux which interpreter should execute the script.
- Variables make scripts reusable.
- User input allows interactive scripts.
- Conditional statements make automation smarter.
- Shell scripting is the foundation of DevOps automation.

---

## Outcome

By the end of Day 16, I can write basic Bash scripts that accept user input, make decisions using conditions, check file existence, and verify Linux service status.
