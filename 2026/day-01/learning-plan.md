# My 90-Day DevOps Learning Plan – Day 01

---

## Who I Am

- **Current level:** College student with basic programming knowledge
- **Background:** Learning DevOps from scratch alongside academics
- **Why DevOps?**  
  Every company runs on infrastructure. I want to be the person who ships, scales, and keeps systems alive — not just write code that works locally. DevOps is where engineering meets real-world impact.

---

## Where I Want to Reach (3 Goals)

1. **Deploy a real application end-to-end** – containerize it with Docker, run it on Kubernetes, and expose it to the internet.
2. **Build a working CI/CD pipeline** – automate code → test → deploy using GitHub Actions or Jenkins.
3. **Get comfortable enough to apply for a DevOps internship or entry-level role** by Day 90.

---

## 3 Core Skills I Will Build

| Skill | Why it matters |
|-------|---------------|
| **Linux + Shell scripting** | Every server runs Linux. If I can't navigate it, I can't debug anything. |
| **Docker + Kubernetes** | The industry standard for packaging and running applications at scale. |
| **CI/CD Pipelines** | Automates the boring stuff and is expected in every DevOps job description. |

---

## Weekly Time Budget

| Day | Time Commitment |
|-----|----------------|
| Mon – Fri | 1 – 1.5 hrs (study + hands-on) |
| Sat – Sun | 3 hrs (projects + revision) |
| **Weekly total** | ~12 hrs |

---

## How I Will Stay Consistent

- **Daily habit:** Open terminal before opening Instagram. No exceptions.
- **Track progress:** Push something to GitHub every single day, even if it's just notes.
- **Learn in public:** Post on LinkedIn 3x per week — progress, not perfection.
- **Stuck?** Spend 20 mins trying to solve it alone, then ask. Don't skip — debug.

| Phase | Days | Focus |
|-------|------|-------|
| Foundation | 1 – 30 | Linux, Shell scripting, Git, Networking basics |
| Core Tools | 31 – 60 | Docker, Kubernetes, AWS basics |
| Real Projects | 61 – 90 | CI/CD pipelines, deploy a full app, build portfolio |
