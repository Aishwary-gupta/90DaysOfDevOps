# 🚀 Day 26 – GitHub CLI (gh): Manage GitHub from Your Terminal

> Part of my **#90DaysOfDevOps** journey.

## 📌 Overview

Today I learned how to manage GitHub directly from the terminal using the **GitHub CLI (`gh`)**.

Instead of switching between the browser and terminal, GitHub CLI allows developers and DevOps engineers to create repositories, manage issues, create pull requests, monitor GitHub Actions, and automate workflows—all from the command line.

This is an essential tool for modern DevOps and CI/CD automation.

---

## 🎯 Objectives

- Install and configure GitHub CLI
- Authenticate with GitHub
- Create and manage repositories
- Work with Issues
- Create and merge Pull Requests
- Explore GitHub Actions commands
- Learn advanced GitHub CLI features

# 📚 Topics Covered

## ✅ Installing GitHub CLI

- Installation
- Authentication
- Checking active account

Commands learned:

```bash
gh auth login
gh auth status
gh --version
```

---

## ✅ Repository Management

Created repositories directly from terminal.

Commands:

```bash
gh repo create
gh repo clone
gh repo list
gh repo view
gh repo delete
gh repo view --web
```

---

## ✅ GitHub Issues

Created and managed GitHub Issues using only terminal.

Commands:

```bash
gh issue create
gh issue list
gh issue view
gh issue close
```

---

## ✅ Pull Requests

Created feature branch

Committed changes

Pushed to GitHub

Created Pull Request

Merged PR

Commands:

```bash
gh pr create
gh pr list
gh pr view
gh pr merge
```

---

## ✅ GitHub Actions

Previewed GitHub Actions from terminal.

Commands:

```bash
gh run list
gh run view
gh workflow list
```

## ✅ Advanced GitHub CLI Features

Explored:

- GitHub API
- Gists
- Releases
- Aliases
- Repository Search

Commands:

```bash
gh api

gh release create
gh alias set
gh search repos
```

# 📖 What I Learned

- GitHub can be managed completely from the terminal.
- GitHub CLI saves time during development.
- Pull Requests can be created without opening GitHub website.
- GitHub CLI integrates perfectly with automation scripts.
- GitHub Actions can be monitored directly from terminal.
- `gh` is an excellent tool for DevOps engineers.
