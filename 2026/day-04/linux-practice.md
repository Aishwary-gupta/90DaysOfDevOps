# Day 04 - Linux Practice: Processes and Services

## Objective

Practice Linux process management, service inspection, and log analysis using commonly used commands.

---

# Process Checks

## 1. List Running Processes

```bash
ps aux
```

**Purpose:** Displays all running processes along with CPU, memory usage, PID, and user.

---

## 2. Monitor Processes

```bash
top
```

**Purpose:** Shows real-time CPU, memory usage, and active processes.

---

## 3. Find a Process

```bash
pgrep ssh
```

**Purpose:** Finds the Process ID (PID) of a running service.

---

# Service Checks

## 1. Check Service Status

```bash
systemctl status ssh
```

**Purpose:** Displays whether the service is active, inactive, or failed.

---

## 2. List Running Services

```bash
systemctl list-units --type=service
```

**Purpose:** Lists all currently loaded services.

---

# Log Checks

## 1. View Service Logs

```bash
journalctl -u ssh
```

**Purpose:** Displays logs generated by the SSH service.

---

## 2. View Recent Logs

```bash
journalctl -u ssh -n 20
```

**Purpose:** Shows the latest 20 log entries for the service.

---

## 3. View System Logs

```bash
tail -n 50 /var/log/syslog
```

**Purpose:** Displays the last 50 lines from the system log file.

---

# Mini Troubleshooting Workflow

1. Check if the service is running.

```bash
systemctl status ssh
```

2. Read the service logs.

```bash
journalctl -u ssh
```

3. Verify the process exists.

```bash
pgrep ssh
```

4. Restart the service if required.

```bash
sudo systemctl restart ssh
```

5. Verify that the service is running again.

```bash
systemctl status ssh
```

---

# Key Learnings

* Processes represent running programs managed by the Linux kernel.
* Services are background applications managed by systemd.
* `systemctl` is used to inspect and manage services.
* `journalctl` is the primary tool for troubleshooting services.
* Reading logs before restarting a service helps identify the root cause of issues.
